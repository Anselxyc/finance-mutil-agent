
# 项目学习顺序

## 阶段一：建立整体认知

1. **通读运行指南**
   - 目标：了解项目组成、模块划分、流程图
   - 关注：项目结构图、执行流程、输入输出示例

2. **跑通一次完整流程**
   - 目标：把文档中的流程和实际运行对上号
   - 操作：
     - 配 `.env`（任选可用的 LLM API）
     - 修改 `mcp_config.py` 中 MCP 服务器路径
     - 执行 `python -m src.main` 并输入示例指令
   - 结果：查看 `logs/` 和 `reports/`，体会系统产物

## 阶段二：理解主业务链路

3. **阅读入口 `src/main.py`**
   - 理解用户输入如何触发 LangGraph 工作流
   - 标记关键函数与数据流转，而非逐行消化

4. **依次阅读 5 个智能体**
   - 目录：`src/agents/`
   - 对每个 agent 关注：输入、调用的工具/LLM、输出形态
   - 目的：明白“谁负责什么 + 信息如何汇总”

5. **掌握 LLM 客户端与状态管理**
   - 文件：`src/utils/llm_clients.py`、`src/utils/state_definition.py`
   - 理解：如何选择不同 LLM、如何组织多 agent 的共享状态

## 阶段三：掌握数据与工具层

6. **研究 MCP 客户端与配置**
   - 文件：`src/tools/mcp_client.py`、`src/tools/mcp_config.py`
   - 关注：MCP 服务器如何启动、如何向工具发请求并获取结果

7. **深入 MCP 服务端（数据源）**
   - 目录：`a-share-mcp-is-just-i-need/src/`
   - 顺序：
     1. 看 `mcp_server.py` 了解暴露的工具
     2. 看 `tools/` 中具体实现，再追到 `baostock_data_source.py`
     3. 理解 `data_source_interface.py` 的抽象
   - 目标：知道每个工具背后如何从 baostock 拉数据

8. **对照日志与报告**
   - 打开 `logs/` 下本地最新运行记录，理解每个阶段的日志含义
   - 打开 `reports/` 下的 Markdown 输出，对照 summary agent 的逻辑

## 阶段四：动手巩固

9. **选择一个“小练习”**
   - 示例：
     - 微调某个 agent 的提示词，观察输出变化
     - 给工具层新增一个简单指标字段，并贯穿到报告
     - 反向追踪一次真实调用，确保每一步都能解释清楚
   - 目的：通过轻量改动把理论转成实践

---

**提示**：如果你在某个阶段停住，可以回到上一步把流程再跑一遍或对照日志。熟悉这个节奏之后，再去考虑扩展功能或接入新的数据源。
