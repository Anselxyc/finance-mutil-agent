# 金融分析智能体系统 - 运行指南

## 📋 项目概述

这是一个基于 **LangGraph** 的多智能体金融分析系统，可以对A股公司进行全面的自动化分析。系统包含以下核心组件：

### 主要模块

1. **Financial-MCP-Agent** - 主分析系统
   - 使用 LangGraph 构建多智能体工作流
   - 包含5个专业分析智能体：
     - 📊 基本面分析智能体（fundamental_agent）
     - 📈 技术面分析智能体（technical_agent）
     - 💰 估值分析智能体（value_agent）
     - 📰 新闻分析智能体（news_agent）
     - 🤖 总结智能体（summary_agent）

2. **a-share-mcp-is-just-i-need** - MCP数据服务器
   - 提供A股市场数据访问接口
   - 基于 baostock 数据源
   - 通过 MCP 协议为智能体提供数据

3. **其他工具脚本**
   - `data_process.py` - 新闻去重处理工具
   - `train_qwen_sentiment.py` / `train_qwen_risk.py` - 模型训练脚本

## 🚀 快速开始

### 第一步：环境准备

#### 1.1 安装 Python 依赖

```bash
# 安装项目依赖
pip install -r requirements.txt
```

主要依赖包括：
- `langgraph==0.6.6` - 工作流框架
- `langchain-openai==0.3.30` - LLM集成
- `baostock==0.8.9` - A股数据源
- `python-dotenv==1.1.1` - 环境变量管理
- `transformers==4.51.3` - 模型框架
- `peft==0.17.0` - 模型微调
- `uv==0.8.12` - 包管理工具（用于MCP服务器）

#### 1.2 安装额外依赖（如果需要）

```bash
# 如果使用新闻去重功能
pip install pandas scikit-learn jieba tqdm numpy

# 如果使用Gemini API
pip install google-genai

# 如果使用OpenAI兼容API
pip install openai
```

### 第二步：配置环境变量

在项目根目录创建 `.env` 文件：

```bash
# 在 Financial-MCP-Agent 目录下创建 .env 文件
cd Financial-MCP-Agent
```

`.env` 文件内容示例：

```env
# Gemini API 配置（如果使用Gemini）
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-1.5-flash

# 或者使用 OpenAI 兼容 API（OpenRouter等）
OPENAI_COMPATIBLE_API_KEY=your_api_key_here
OPENAI_COMPATIBLE_BASE_URL=https://api.openrouter.ai/api/v1
OPENAI_COMPATIBLE_MODEL=your_model_name_here
```

**重要说明**：
- 至少需要配置一个LLM API（Gemini 或 OpenAI兼容API）
- 如果使用 Gemini，可能需要美国节点VPN（有地理位置限制）
- 如果使用 OpenRouter，需要注册账号并获取API密钥

### 第三步：配置MCP服务器路径

编辑 `Financial-MCP-Agent/src/tools/mcp_config.py`，修改MCP服务器路径：

```python
SERVER_CONFIGS = {
    "a_share_mcp_v2": {  
        "command": "uv", 
        "args": [
            "run",  
            "--directory",
            r"E:\股票分析agent\鑲＄エ鍒嗘瀽agent\Finance\a-share-mcp-is-just-i-need",  # 修改为你的实际路径
            "python",
            "mcp_server.py"
        ],
        "transport": "stdio",
    }
}
```

**注意**：将路径修改为你本地 `a-share-mcp-is-just-i-need` 目录的绝对路径。

### 第四步：运行系统

#### 方式1：交互式运行（推荐）

```bash
cd Financial-MCP-Agent
python -m src.main
```

系统会显示欢迎界面，然后提示你输入分析需求，例如：
- `分析嘉友国际`
- `帮我看看比亚迪(002594)这只股票怎么样`
- `分析一下宁德时代的财务状况`
- `603871 这个股票值得买吗？`

#### 方式2：命令行参数运行

```bash
cd Financial-MCP-Agent
python -m src.main --command "分析嘉友国际(603871)"
```

## 🔧 工作流程说明

系统执行流程：

```
用户输入查询
    ↓
提取股票代码/公司名称
    ↓
启动4个并行分析智能体：
├── 基本面分析智能体 → 分析财务状况、盈利能力
├── 技术面分析智能体 → 分析价格趋势、技术指标
├── 估值分析智能体 → 分析估值水平
└── 新闻分析智能体 → 分析新闻情感和风险
    ↓
总结智能体整合所有分析结果
    ↓
生成最终分析报告
    ↓
保存报告到 reports/ 目录
```

## ⚙️ 配置详解

### MCP服务器配置

MCP（Model Context Protocol）服务器提供数据访问接口。系统通过MCP协议调用以下工具：

- 股票市场数据（K线、基本信息等）
- 财务报告数据
- 指数数据
- 市场概览
- 宏观经济数据
- 新闻爬虫

### LLM客户端配置

系统支持两种LLM客户端：

1. **Gemini客户端**（默认）
   - 需要 `GEMINI_API_KEY`
   - 可能需要VPN（地理位置限制）

2. **OpenAI兼容客户端**
   - 需要 `OPENAI_COMPATIBLE_API_KEY`、`OPENAI_COMPATIBLE_BASE_URL`、`OPENAI_COMPATIBLE_MODEL`
   - 支持 OpenRouter、本地部署的兼容API等

## 🐛 常见问题

### 1. MCP服务器连接失败

**问题**：提示无法连接到MCP服务器

**解决方案**：
- 检查 `mcp_config.py` 中的路径是否正确
- 确保已安装 `uv`：`pip install uv`
- 尝试手动运行MCP服务器测试：
  ```bash
  cd a-share-mcp-is-just-i-need
  uv run python mcp_server.py
  ```

### 2. API密钥错误

**问题**：提示API密钥未找到或无效

**解决方案**：
- 检查 `.env` 文件是否在 `Financial-MCP-Agent` 目录下
- 确认环境变量名称正确（区分大小写）
- 验证API密钥是否有效

### 3. Gemini API地理位置限制

**问题**：Gemini API返回地理位置错误

**解决方案**：
- 使用美国节点的VPN
- 或切换到OpenRouter等OpenAI兼容API

### 4. 股票代码识别失败

**问题**：系统无法识别股票代码或公司名称

**解决方案**：
- 尽量使用标准格式：`公司名称(股票代码)`，如 `嘉友国际(603871)`
- 或直接使用股票代码：`603871`
- 确保股票代码格式正确（5-6位数字）

### 5. 数据获取失败

**问题**：无法获取股票数据

**解决方案**：
- 检查网络连接
- 确认baostock服务是否正常
- 检查股票代码是否正确（6开头=上海，0/3开头=深圳）

## 📝 注意事项

1. **数据源**：系统依赖 baostock 提供A股数据，确保网络连接正常
2. **API费用**：使用LLM API会产生费用，注意控制使用量
